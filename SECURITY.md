# Política de Seguridad

## Reportar vulnerabilidades

Si encontrás una vulnerabilidad de seguridad, **no la publiques como issue público**. Contactame directamente vía Discord: **imsh4d0w_**.

Incluí en tu reporte:

- Descripción de la vulnerabilidad
- Pasos para reproducirla
- Impacto potencial
- Cómo la detectaste

Responderé en un plazo razonable y trabajaré contigo para resolverla.

## Qué NO es una vulnerabilidad

- Falsos positivos o negativos en el análisis de amenazas
- Límites de API de VirusTotal o SightEngine
- Comportamiento esperado del anti-spam o rate limiting
- Problemas de rendimiento no críticos

## Qué SÍ es una vulnerabilidad

- Acceso no autorizado a datos de otros servidores
- Bypass del sistema de permisos
- Inyección de código o comandos
- Exposición de tokens o credenciales
- SSRF o ataques de red
- Escalada de privilegios

## Responsable disclosure

Si reportás una vulnerabilidad de forma responsable, serás reconocido en los agradecimientos (a menos que prefieras anonimato).
